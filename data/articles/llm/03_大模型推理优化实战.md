## 2. 架构深度对比

### 2.1 vLLM: PagedAttention 革命

**核心创新**:
```
传统注意力机制：
- 连续内存分配
- 显存碎片化严重
- 并发请求数受限

PagedAttention:
- 分页内存管理（类似操作系统虚拟内存）
- 内存利用率提升 80%
- 并发请求数提升 4-8 倍
```

**架构优势**:
- ✅ 高吞吐量（适合多用户并发）
- ✅ 动态工作负载适应性强
- ✅ 与 HuggingFace 生态无缝集成
- ✅ 支持多种 GPU（NVIDIA/AMD/Intel）

**性能指标** (A100 80GB, LLaMA-3-70B):
```
- 吞吐量：2800 tokens/s (batch=256)
- 显存利用率：92%
- 并发请求：512 个
- 时间到首 token: 120ms
```

### 2.2 TensorRT-LLM: 硬件级优化

**核心创新**:
```
编译时优化：
- 算子融合（矩阵乘法 + 激活函数）
- 硬件特定内核选择
- 内存布局调优
- 量化感知训练（FP8/FP4/INT4）

运行时优化：
- 自定义注意力内核
- 动态批处理
- 投机性解码
```

**架构优势**:
- ✅ 极致延迟优化（单请求场景）
- ✅ NVIDIA GPU 深度优化（A100/H100/L40S）
- ✅ 量化支持最全面（FP8/FP4/INT4 AWQ/INT8 SmoothQuant）
- ✅ 多 GPU 分布式推理成熟

**性能指标** (H100 80GB, LLaMA-3-70B, FP8 量化):
```
- 吞吐量：3500 tokens/s (batch=256)
- 显存利用率：95%
- 并发请求：384 个
- 时间到首 token: 78ms
- 编译时间：25 分钟
```

## 4. 生产部署实战

### 4.1 vLLM 部署

**快速启动**:
```bash
# 安装
pip install vllm

# 单 GPU 启动
python -m vllm.entrypoints.api_server \
    --model meta-llama/Llama-3-70b-instruct \
    --host 0.0.0.0 \
    --port 8000 \
    --tensor-parallel-size 1 \
    --max-model-len 4096 \
    --gpu-memory-utilization 0.9

# 多 GPU 启动 (8 卡)
python -m vllm.entrypoints.api_server \
    --model meta-llama/Llama-3-70b-instruct \
    --tensor-parallel-size 8 \
    --max-num-batched-tokens 8192 \
    --enable-prefix-caching
```

**Docker 部署**:
```yaml
version: '3.8'
services:
  vllm:
    image: vllm/vllm-openai:latest
    deploy:
      resources:
        reservations:
          devices:
            - driver: nvidia
              device_ids: ['0','1','2','3']
              capabilities: [gpu]
    ports:
      - "8000:8000"
    volumes:
      - ./models:/root/.cache/huggingface
    command:
      - --model
      - meta-llama/Llama-3-70b-instruct
      - --tensor-parallel-size
      - "4"
      - --gpu-memory-utilization
      - "0.95"
```

### 4.2 TensorRT-LLM 部署

**模型编译**:
```bash
# 安装
pip install tensorrt-llm

# 构建模型引擎 (Llama-3-70B, FP8)
trtllm-build \
    --checkpoint_dir /path/to/llama-3-70b \
    --output_dir /path/to/engine \
    --max_batch_size 256 \
    --max_input_len 4096 \
    --max_output_len 2048 \
    --use_fp8 \
    --gemm_plugin fp8 \
    --world_size 8
```

**推理服务**:
```python
import tensorrt_llm
from tensorrt_llm.runtime import ModelRunner

# 加载引擎
runner = ModelRunner.from_engine(engine_path)

# 推理
input_ids = tokenizer.encode("Hello, how are you?")
output = runner.generate(input_ids, max_new_tokens=256)
print(tokenizer.decode(output))
```

### 4.3 混合部署策略

**场景**: 单模型日请求量>2 亿 tokens

**架构**:
```
负载均衡器 (Nginx)
    ↓
┌───────────────────┬───────────────────┐
│   vLLM (8 卡)      │  TensorRT-LLM (4 卡) │
│   批处理请求       │   低延迟请求        │
│   吞吐量优先       │   延迟优先          │
└───────────────────┴───────────────────┘
```

**路由规则**:
```nginx
# Nginx 配置
location /api/generate {
    if ($http_x_priority = "low_latency") {
        proxy_pass http://tensorrt_backend;
    }
    default {
        proxy_pass http://vllm_backend;
    }
}
```

**效果**:
- 低延迟请求 P99: 150ms (vs 320ms)
- 整体吞吐量：提升 25%
- 成本节省：30% (减少 GPU 数量)

## 5. 量化优化

### 5.1 量化方案对比

| 方案 | 精度 | 压缩比 | 速度提升 | 质量损失 | 适用场景 |
|------|------|--------|---------|---------|---------|
| FP16 | 16bit | 1x | 基准 | 无 | 生产基准 |
| FP8 | 8bit | 2x | 1.8x | <1% | 推荐默认 |
| INT8 | 8bit | 2x | 1.6x | 1-2% | 兼容性优先 |
| INT4 AWQ | 4bit | 4x | 2.5x | 2-3% | 资源受限 |
| INT4 GPTQ | 4bit | 4x | 2.3x | 2-4% | 通用场景 |

### 5.2 vLLM 量化

```bash
# AWQ 量化模型
python -m vllm.entrypoints.api_server \
    --model TheBloke/Llama-3-70B-AWQ \
    --quantization awq \
    --dtype float16

# GPTQ 量化
python -m vllm.entrypoints.api_server \
    --model TheBloke/Llama-3-70B-GPTQ \
    --quantization gptq \
    --gptq_model
```

### 5.3 TensorRT-LLM 量化

```bash
# FP8 量化 (推荐 H100)
trtllm-build \
    --checkpoint_dir /path/to/llama-3-70b \
    --output_dir /path/to/fp8-engine \
    --use_fp8 \
    --gemm_plugin fp8

# INT4 AWQ 量化
trtllm-build \
    --checkpoint_dir /path/to/llama-3-70b-awq \
    --output_dir /path/to/int4-engine \
    --use_int4_awq
```

## 9. 总结

**vLLM 适合**:
- ✅ 高并发、多租户场景
- ✅ 快速原型和迭代
- ✅ 多模型热切换
- ✅ 异构 GPU 环境

**TensorRT-LLM 适合**:
- ✅ 低延迟、单模型大规模
- ✅ NVIDIA 最新 GPU (H100/Blackwell)
- ✅ 成本敏感、追求极致性能
- ✅ 固定模型长期部署

**最佳实践**: 混合部署，vLLM 处理批处理请求，TensorRT-LLM 处理低延迟请求。
